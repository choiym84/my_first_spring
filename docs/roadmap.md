# Roadmap

This file is the source of truth for upcoming learning issues.

## Current Phase
Board project with Spring MVC, Thymeleaf, Spring Data JPA, and H2.

## Issue Backlog

### Issue 20: Create `Post` Entity
Status: Done

Goal:
- Define the board post domain model.

Acceptance criteria:
- `Post` is a JPA entity.
- It has `id`, `title`, `content`, `writer`, `createdAt`.
- `id` is generated by JPA.
- `createdAt` is set when the entity is created.

### Issue 21: Create `PostRepository`
Status: In progress

Goal:
- Create a Spring Data JPA repository for posts.

Acceptance criteria:
- `PostRepository` extends `JpaRepository<Post, Long>`.
- It has `findAllByOrderByCreatedAtDesc()`.
- It does not use unnecessary `@Primary`.

### Issue 22: Create `PostService`
Status: Todo

Goal:
- Add business logic for writing and reading posts.

Acceptance criteria:
- `write(Post post)` saves a post.
- `findPosts()` returns posts by latest first.
- Empty title/content validation is handled.

### Issue 23: Create Post Write Page
Status: Todo

Goal:
- Add a page for writing posts.

Acceptance criteria:
- `GET /posts/new` shows a form.
- `POST /posts/new` saves the post.
- Success redirects to `/posts`.
- Validation failure returns to the form with an error message.

### Issue 24: Create Post List Page
Status: Todo

Goal:
- Show posts in a board list.

Acceptance criteria:
- `GET /posts` shows post list.
- Posts are sorted by latest first.
- Each row shows id, title, writer, created time.

### Issue 25: Create Post Detail Page
Status: Todo

Goal:
- Show one post.

Acceptance criteria:
- `GET /posts/{id}` shows one post.
- Missing post is handled gracefully.

### Issue 26: Add Post Edit Feature
Status: Todo

Goal:
- Edit title and content.

Acceptance criteria:
- `GET /posts/{id}/edit` shows edit form.
- `POST /posts/{id}/edit` updates the post.
- `Post` uses a meaningful method such as `update(title, content)`.

### Issue 27: Add Post Delete Feature
Status: Todo

Goal:
- Delete a post.

Acceptance criteria:
- A post can be deleted by id.
- After deletion, user is redirected to `/posts`.

## Working Rule
- One issue should match one learning step.
- One issue should be small enough to finish in one focused session.
- Each completed issue should have at least one commit.
- If a concept caused confusion, add it to `docs/troubleshooting.md`.
